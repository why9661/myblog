Golang-String

2021-01-18 2 分钟阅读 #Golang#Golang

Go使用UTF-8作为默认编码方式。

String类型的数据结构

每一个字符串在运行时都会使用如下的数据结构

1type StringHeader struct {
2	Data uintptr //指向字节数组的指针
3	Len  int	 //数组的大小（字节数）
4}

因为String类型指向的底层字节数组是只读的，所以不能像这样修改字符串：

1fmt.Println(str[0]) // √
2str[0] = 'h' // ×

要修改字符串，可以给字符串赋新值。

1str = "new string"

使用unsafe包对字符串原地修改

 1func main() {
 2	s1 := "Hello World!"
 3	s2 := "why"
 4
 5	tmp1 := (*reflect.StringHeader)(unsafe.Pointer(&s1))
 6	tmp2 := (*reflect.StringHeader)(unsafe.Pointer(&s2))
 7	tmp1.Data = tmp2.Data
 8	tmp1.Len = tmp2.Len
 9
10	fmt.Println(s1) //why
11}

字符串遍历

  • 普通for循环遍历

    遍历的是单个字节（byte）

  • for range遍历

    遍历的是单个字符（rune）

1a := "H"
2b := 'H' 
3var c byte
4c = 'H'
5fmt.Printf("%T\n", a) //string
6fmt.Printf("%T\n", b) //int32(rune)
7fmt.Printf("%T\n", c) //uint8(byte)

反引号&双引号

使用双引号声明的字符串和其他语言中的字符串没有太多的区别，它只能用于单行字符串的初始化，如果字符串内部出现双引号，需要使用 \转义 符号避免编译器的解析错误，而反引号声明的字符串可以摆脱单行的限制。当使用反引号时，因为双引号不再负责标记字符串的开始和结束，可以在字符串内部直接使用 "，在遇到需要手写 JSON 或者其他复杂数据格式的场景下非常方便。

1json := `{"author": "draven", "tags": ["golang"]}`

String的常用操作

标准库strings和strconv包含了很多String的常用操作。

注意事项

String类型可以和[]byte和[]rune类型直接转换（开销并不低，会发生底层数组的拷贝）。

通过unsafe包实现[]byte和string类型的转换

 1func byteToString(b []byte) string {
 2	header := (*reflect.SliceHeader)(unsafe.Pointer(&b))
 3
 4	newHeader := reflect.StringHeader{
 5		Data: header.Data,
 6		Len: header.Len,
 7	}
 8
 9	return *(*string)(unsafe.Pointer(&newHeader))
10}
11
12func stringToByte(s string) []byte {
13	header := (*reflect.StringHeader)(unsafe.Pointer(&s))
14
15	newHeader := reflect.SliceHeader{
16		Data: header.Data,
17		Len: header.Len,
18		Cap: header.Len,
19	}
20
21	return *(*[]byte)(unsafe.Pointer(&newHeader))
22}
23
24func main() {
25	b1 := []byte{65, 66, 67}
26	s1 := byteToString(b1)
27	fmt.Println(s1) // ABC
28
29	s2 := "ABC"
30	b2 := stringToByte(s2)
31	fmt.Println(b2) //[65 66 67]
32}
