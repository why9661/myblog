Golang-闭包

2021-04-27 2 分钟阅读 #Golang#Golang

Function Value

在Go语言中，函数是头等对象，可以作为参数传递，可以作为返回值返回，也可赋值给变量。Go语言称这样的参数、返回值、变量为Function Value。

Function Value是一个指针，指向一个runtime.funcval结构体：

1type funcval struct {
2	fn uintptr 
3}

这个结构体持有一个指针fn，指向函数指令入口。

例子

函数A被赋值给变量f1和f2。

编译器会做出优化，在只读数据段分配一个funcval结构体，f1和f2共用这个结构体，结构体中的fn指向函数A的指令入口。

 1func A() int {
 2	i := 1
 3	return i
 4}
 5
 6func main() {
 7	f1 := A
 8	f2 := A
 9    fmt.Println(f1())
10    fmt.Println(f2())
11}

闭包

闭包可以简单理解为：函数+引用环境。

例子

返回的匿名函数引用了外部的局部变量x，因此返回的这个函数就是一个闭包。

闭包是有捕获列表的Function Value。

执行到f1:=A()时，函数A会在堆上分配一个funcval结构体，同时拷贝x的值到捕获列表，再将这个funcval的起始地址返回给f1。

执行到f2:=A()时，函数A会在堆上再分配一个funcval结构体，同时拷贝x的值到捕获列表，再将这个funcval的起始地址返回给f2。

 1func A() func() int {
 2	x := 1
 3	return func() int { //闭包
 4		return x
 5	}
 6}
 7
 8func main() {
 9	f1 := A()
10	f2 := A()
11	fmt.Println(f1()) //1
12	fmt.Println(f2()) //1
13}

以上是被捕获的变量不会被修改的情况，直接拷贝值到捕获列表就ok。

接下来看被捕获的变量会被修改的情况：

捕获局部变量。

执行到f1:=A()时，函数A会在堆上分配一个funcval结构体，由于被捕获的变量被外部引用且被修改，x会逃逸到堆上，并且捕获列表存x的地址，然后将这个funcval的起始地址返回给f1。

执行到f2:=A()时，函数A会在堆上再分配一个funcval结构体，由于被捕获的变量被外部引用且被修改，x会逃逸到堆上，并且捕获列表存x的地址，然后将这个funcval的起始地址返回给f2。

 1func A() func() int {
 2	x := 1
 3	return func() int { //闭包
 4		x++
 5		return x
 6	}
 7}
 8
 9func main() {
10	f1 := A()
11	f2 := A()
12	fmt.Println(f1()) //2
13	fmt.Println(f2()) //2
14	fmt.Println(f1()) //3
15	fmt.Println(f2()) //3
16}

闭包引起的一些问题

 1func main() {
 2	for i := 0; i < 5; i++ {
 3		defer func() {
 4			fmt.Println(i)
 5		}()
 6	}
 7}
 8//输出
 9//5
10//5
11//5
12//5
13//5

解决方法：

 1func main() {
 2	for i := 0; i < 5; i++ {
 3        x := i
 4		defer func() {
 5			fmt.Println(x)
 6		}()
 7	}
 8}
 9
10func main() {
11	for i := 0; i < 5; i++ {
12		defer func(x int) {
13			fmt.Println(x)
14		}(i)
15	}
16}
