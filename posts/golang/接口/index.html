类型元数据

不管是内置类型还是自定义类型，都有自己的类型元数据。

每种类型元数据都是由runtime._type和其它描述信息组成（如果是自定义类型，还会通过uncommontype记录类型元数据的信息）。

数据结构

接口的底层数据结构有两种runtime.iface和runtime.eface。

eface表示不包含任何方法的空接口。

iface表示包含方法的接口。

1type eface struct {
2   _type *_type        //动态类型元数据
3   data  unsafe.Pointer//动态值
4}

 1type iface struct {
 2   tab  *itab		  //动态类型元数据、接口类型元数据（包含接口要求的方法列表）等
 3   data unsafe.Pointer//动态值
 4}
 5
 6type itab struct { // 32 字节
 7	inter *interfacetype
 8	_type *_type
 9	hash  uint32
10	_     [4]byte
11	fun   [1]uintptr
12}

类型断言

类型断言作用于接口值（包括空接口和非空接口）。

而断言的类型可以是具体类型或非空接口类型。

语法：接口值.(断言类型)

因此类型断言可以分为以下4种情况：

  • 空接口.(具体类型) A.(B)

    判断A的动态类型是否和具体类型B的类型元数据相同，相同则断言成功，否则断言失败。

    断言成功返回A的动态值，类型当然是具体类型B，ok=true

    失败会触发panic（使用两个值接收断言的结果不会触发panic，返回B的类型零值，ok=false）。

  • 非空接口.(具体类型) A.(B)

    判断A的动态类型是否和具体类型B相同，相同则断言成功，否则断言失败。

    断言成功返回A的动态值，类型当然是具体类型B。

    失败会触发panic（使用两个值接收断言的结果不会触发panic，返回B的零值）。

  • 空接口.(非空接口类型) A.(B)

    判断A的动态类型是否实现了非空接口B，实现则断言成功，否则断言失败。

    断言成功返回一个B类型的接口变量，动态值与A相同，动态类型元数据与B相同。

    失败会触发panic（使用两个值接收断言的结果不会触发panic，返回B的零值）。

  • 非空接口.(非空接口类型) A.(B)

    判断A的动态类型是否实现了非空接口B，实现则断言成功，否则断言失败。

    断言成功返回一个B类型的接口变量，动态值与A相同，动态类型元数据与B相同。

    失败会触发panic（使用两个值接收断言的结果不会触发panic，返回B的零值）。

反射

反射的作用是把类型元数据暴露给用户使用。

两个重要方法：reflect.TypeOf() reflect.ValueOf()

两个重要类型：reflect.Value reflect.Type

常用方法

reflect.Value.Kind()和reflect.Type.Kind()

返回具体类型，如reflect.Struct、reflect.Ptr、reflect.String等。

reflect.Type.Name()

返回自定义类型在包中的名称

 1type student struct{
 2	name string
 3	age int
 4}
 5
 6func main() {
 7	s1 := student{
 8		name: "why",
 9		age: 18,
10	}
11
12	t := reflect.TypeOf(s1)
13	fmt.Println(t.Name()) //student
14}

**reflect.Value.Interface()**返回一个空接口，动态类型和动态值与v相同。

通过反射修改值

通过反射调用方法

 1type student struct{
 2	name string
 3	age int
 4}
 5
 6func (s student) Print() {
 7	fmt.Println(s.name)
 8	fmt.Println(s.age)
 9}
10
11func main() {
12	s := student{
13		name: "why",
14		age: 18,
15	}
16    
17	t := reflect.TypeOf(s)
18	v := reflect.ValueOf(s)
19    
20	m1, _ := t.MethodByName("Print")
21	m2 := v.MethodByName("Print")
22    
23    m1.Func.Call([]reflect.Value{reflect.ValueOf(student{"why", 18})})
24	m2.Call([]reflect.Value{})
25}

针对结构体的方法

  • **reflect.Type.NumField()**返回结构体的字段数目

  • **reflect.Type.Field()**返回结构体的第i个字段(reflect.StructField)，通过StructField可以获取该字段的名字、类型、tag等信息

  • **reflect.StructTag.Lookup()、reflect.StructTag.Get()**获取tag信息

     1type student struct{
 2	name string `test:"test"`
 3	age int
 4}
 5
 6func main() {
 7	s := student{
 8      name: "why",
 9      age: 18,
10	}
11	tag, ok := reflect.TypeOf(s).Field(0).Tag.Lookup("test")
12	//tag := reflect.TypeOf(s).Field(0).Tag.Get("test")
13	fmt.Println(tag, ok)
14}

注意事项

  • 如果一个结构体嵌套的匿名结构体实现了某个接口，那么这个结构体也实现了该接口。

  • 如果一个结构体实现了某个接口，那么该结构体类型和相应的指针类型都可以被声明为该接口类型。

  • 如果一个结构体指针实现了某个接口，那么只有该结构体相应的指针类型才能被声明为该接口类型。

  • 空接口类型并不等于任意类型

    1func main() {
2	var i *int
3	var e interface{}
4    fmt.Println(i == nil) //true 指针类型的零值是nil
5	fmt.Println(e == nil) //true 接口类型的零值是nil
6	e = i 				  //隐式的类型转换
7	fmt.Println(e == nil) //false 空接口由两部分组成，将i赋值给e时，动态类型部分是*int
8}
