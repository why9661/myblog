Protobuf基础

2021-06-10 3 分钟阅读 #Protobuf#Protobuf

简介

ProtoBuf 即 Protocol Buffers，是一种轻便高效的结构化数据存储格式，与语言、平台无关，可扩展可序列化。ProtoBuf 性能和效率大幅度优于 JSON、XML 等其他的结构化数据格式。ProtoBuf 是以二进制方式存储的，占用空间小，但也带来了可读性差的缺点。

安装

https://github.com/protocolbuffers/protobuf/releases

加入到环境变量。

protoc –version验证是否安装成功。

在 Golang 中使用 protobuf，还需要安装 protoc-gen-go，这个工具用来将 .proto 文件转换为 Go代码。

1go get -u github.com/golang/protobuf/protoc-gen-go

这会自动安装到$GOPATH/bin下，同样加入到环境变量。

使用

一个小例子

创建.proto文件

1syntax = "proto3"; 
2package pb;
3option go_package = "./";
4
5message Person {
6  string name = 1;
7  int32 age = 2;
8  repeated string hobbies = 3;
9}

根据.proto文件生成代码

1protoc --go_out=$DST_DIR $SRC_DIR/xxx.proto

–go_out表示生成Go代码

$DST_DIR表示生成代码的目录

$SRC_DIR/xxx.proto表示具体的proto文件

1...
2type Person struct {
3   ...
4
5   Name    string   `protobuf:"bytes,1,opt,name=name,proto3" json:"name,omitempty"`
6   Age     int32    `protobuf:"varint,2,opt,name=age,proto3" json:"age,omitempty"`
7   Hobbies []string `protobuf:"bytes,3,rep,name=hobbies,proto3" json:"hobbies,omitempty"`
8}
9...

接下来就可以在代码中使用了

 1func main() {
 2   person := pb.Person{
 3      Name: "why",
 4      Age: 24,
 5      Hobbies: []string{"A", "B", "C"},
 6   }
 7
 8   data, err := proto.Marshal(&person)
 9   if err != nil {
10      log.Fatal("Marshaling error: ", err)
11   }
12
13   newPerson := &pb.Person{}
14   err = proto.Unmarshal(data, newPerson)
15   if err != nil {
16      log.Fatal("Unmarshaling error: ", err)
17   }
18
19   fmt.Printf("%+v", newPerson) //name:"why"  age:24  hobbies:"A"  hobbies:"B"  hobbies:"C"
20
21}

基础知识

具体使用参考官方文档：https://developers.google.com/protocol-buffers/docs/proto3#simple

注释使用//…和/*…*/

如果要使用proto3必须在第一行syntax="proto3"，否则会编译器会使用proto2

一个.proto文件中可以定义多个message

在一个message里字段编号是唯一的，且一旦使用不能再更改。编号范围为1~2^29-1，其中19000~19999不能使用。

字段默认的修饰符是singular。repeated表示字段可重复（生成go代码后为切片）。

枚举类型（Enumerations）

枚举类型适用于提供一组预定义的值，选择其中一个。

必须有0值，且为第一个。

 1message Person {
 2  string name = 1;
 3  int32 age = 2;
 4  repeated string hobbies = 3;
 5  enum Gender {
 6    MALE = 0;
 7    FEMALE = 1;
 8  }
 9  Gender gender = 4;
10}

 1func main() {
 2	person := pb.Person{
 3		Name: "why",
 4		Age: 24,
 5		Hobbies: []string{"A", "B", "C"},
 6		Gender: 0,
 7	}
 8
 9	data, err := proto.Marshal(&person)
10	if err != nil {
11		log.Fatal("Marshaling error: ", err)
12	}
13
14	newPerson := &pb.Person{}
15	err = proto.Unmarshal(data, newPerson)
16	if err != nil {
17		log.Fatal("Unmarshaling error: ", err)
18	}
19
20	fmt.Println(newPerson.GetGender()) //MALE
21}

使用其它消息类型

可以使用其它message类型作为字段。

1message SearchResponse {
2  repeated Result results = 1;
3}
4
5message Result {
6  string url = 1;
7  string title = 2;
8  repeated string snippets = 3;
9}

如果定义在其他文件中，可以先导入然后使用

1import "myproject/other_protos.proto";

也可以直接嵌套

1message SearchResponse {
2  message Result {
3    string url = 1;
4    string title = 2;
5    repeated string snippets = 3;
6  }
7  repeated Result results = 1;
8}

定义服务

如果消息类型是用来远程通信的(Remote Procedure Call, RPC)，可以在 .proto 文件中定义 RPC 服务接口。

如下面这个例子，定义了服务SearchService并提供了Search接口，入参是SearchRequest，返回类型是SearchResponse。

 1message SearchRequest {
 2  ...
 3}
 4
 5message SearchResponse {
 6 ...
 7}
 8
 9service SearchService {
10  rpc Search(SearchRequest) returns (SearchResponse);
11}
